public static int[] specialSort(int[] arr) {
	int front = 0;
	int back = arr.length - 1;
	int curr = 0;
	while (curr <= back) {
		if (arr[curr] < 1) {
		swap(arr, curr, front);
		front += 1;
		curr += 1;
	} else if (arr[curr] > 1) {
		swap(arr, curr, back);
		back -= 1;
	} else {
		curr += 1;
	}
	}
}

由于 curr 是从左往右扫描的，它就像一个“侦察兵”，扫描过的地方都被它看穿了。

    在 front 之前的，全是 0。

    在 front 到 curr - 1 之间的，全是 1。

当你发现 arr[curr] 是 0 并准备和 arr[front] 交换时，只有两种情况：
    情况 A：curr == front
    也就是 curr 和 front 指向同一个位置（比如数组开头全是 0）。这时候自己换自己，换完之后这个位置当然是排好的 0，curr 放心往前走。
    情况 B：curr > front
    这意味着 front 指向的是一个已经确定了的 1（因为所有被 curr 扫描过的 1 都会被留在 front 和 curr 之间）。
    当你把 curr 位置的 0 换到 front 时，换到 curr 位置的那个数百分之百是 1。
既然你已经百分之百确定换回来的数是 1 了，那还有必要在下一轮重新检查它吗？不需要了。所以 curr 可以潇洒地 += 1。



当你发现数据种类变多（比如 4 种、10 种、甚至 100 种固定整数），我们就不再折腾指针了，而是祭出另一种大杀器：计数排序 (Counting Sort)。
计数排序的逻辑（极其简单）：
    数一数：开一个长度为 4 的数组 count，遍历一遍原数组。
        看到 0，count[0]++
        看到 1，count[1]++ ...
    填回去：根据 count 里的结果，直接把数组填满。
        比如 count[0] 是 5，就把前 5 个位置全填 0。
为什么 0, 1, 2 时不用计数排序？
因为三指针算法是 In-place（原地） 且 One-pass（一次遍历） 的。计数排序通常需要额外的空间，且要遍历两遍（数一遍，填一遍）。在只有 3 种数据时，三指针更极致、更优雅。
