实际上，计算机并不是真的在一个格子里既存了符号又存了数字。它是通过**“索引映射（Mapping）”**这种间接的方式来完成的。
1. 数组里到底存了什么？
在代码实现中，那个所谓的 Counts 或 Starting Points 其实只是一个普通的整型数组（例如 int[]）。
    内存里只有数字： counts 数组在内存里看起来就是 [3, 2, 4, 3]。
    秘密在于“下标”： 我们在写程序时，人为约定了一个规则（映射函数）：
        下标 0 → 代表 ♣ (梅花)
        下标 1 → 代表 ♠ (黑桃)
        下标 2 → 代表 ♡ (红心)
        下标 3 → 代表 ♢ (方片)
所以，当你问“方片怎么在 0 的位置”时：
其实在 Starting Points 数组里，方片的计数值存放在 下标 3 的位置。而这个下标 3 对应的值是 10（看你图中红字下标 3 对应的 Starting Point 是 10）。这代表：在最终的排好序的表里，第一个方片应该放在第 10 号格子。
2. 它是怎么“指定”对应关系的？
在程序运行前，我们会建立一个查找表（Lookup Table）。你可以把它想象成一个翻译官。
当程序扫描原始数据（比如看到一个 ♢ Glaser）时，它会做以下动作：
    查询翻译官： “方片对应的数字是多少？” → 翻译官回答：“是 3”。
    访问数组： 程序直接去访问 counts[3]（即数组的第 4 个位置）。
    操作数据： 把 counts[3] 里的数字加 1。
3. 为什么图片里看起来像两列？
图片里左边那一列符号（梅花、黑桃...）其实是给人类看的注释，方便我们理解这个格子里存的是谁的计数。
在真实的计算机内存里：
    Alphabet 数组：存的是 ['♣', '♠', '♥', '♦']。
    Counts 数组：存的是 [3, 2, 4, 3]。
如果你想把方片放在数组的 0 号位置，你只需要修改 Alphabet 的顺序：
把 Alphabet 定义为 {♦, ♥, ♠, ♣}。
这样，翻译官就会告诉程序：“方片对应的数字是 0”。于是程序就会去操作 counts[0] 了。
4. 总结：如何实现“对号入座”
这就是计数排序高效的原因：它利用了数组下标的随机访问特性。
    Input: ♢ (方片)
    Mapping: indexOf('♦') → 返回 3
    Access: StartingPoints[3] → 得到 10
    Result: 把 Glaser 放到 SortedArray[10] 的位置。
一句话总结： 数组不存 Alphabet，它只存数字；Alphabet 是一把钥匙，告诉程序该开哪一把锁（访问哪一个数组下标）。
