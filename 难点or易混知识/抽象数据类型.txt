概念				这里的“抽象”意思				举例					对应现实
ADT (抽象数据类型)	我要什么功能？					List, Set, Map			微波炉的外壳和按钮 (你能按按钮加热，不用管里面)
数据结构 (实现)		怎么存数据？					Array, Linked List, Tree	微波炉内部的磁控管 (具体的零件)
算法				怎么处理数据？					排序, 二分查找, 遍历		加热程序的逻辑 (先大火3分钟，再转小火)
ADT (抽象数据类型): 是定义（也就是“它能做什么”）。
Data Structure (数据结构): 是实现（也就是“它怎么存数据”）。
Algorithm (算法): 是步骤（也就是“怎么操作这些数据来完成任务”）。

1. 什么是 ADT（抽象数据类型）？
核心词：规则、接口、承诺。
ADT 定义了逻辑上的行为，但完全不关心内部是怎么做到的。
    例子（汽车）：
        ADT 定义： 汽车是一个有方向盘、油门、刹车的东西。
        操作： 你踩油门，车就走；你转方向盘，车就转。
        抽象在哪？ 你根本不需要知道它是“燃油发动机”还是“电动马达”，也不需要知道汽油是怎么燃烧的。只要它能走能停，它就符合“汽车 ADT”。
    例子（编程中的 List ADT）：
        定义： 这是一个有序的序列。
        操作： add(x) (添加), get(i) (获取第i个), remove(i) (删除)。
        不管实现： 至于它是用数组存的，还是用链表连起来的，ADT 不在乎。
2. 什么是数据结构（Data Structure）？
核心词：存储方式、物理实现。
这是 ADT 在计算机内存里的具体写法。你提到的“数组”和“链表”通常属于这一层。
    回到汽车：
        如果要造一辆车，你必须决定是用 V8发动机 还是 锂电池。这就是“数据结构”。
    回到编程：
        你要实现一个 List (ADT)，你可以选择用 Array (数组) 来存，也可以用 Linked List (链表) 来存。
        Array 和 Linked List 是数据结构。
3. 什么是算法（Algorithm）？
核心词：步骤、逻辑、流程。
算法是用来操作数据结构，从而实现 ADT 功能的具体步骤。
    回到汽车：
        当你踩下刹车时，ABS系统如何点刹防止抱死？那个控制刹车片一松一紧的逻辑程序，就是算法。
    回到编程：
        比如要在 List 里查找一个数字。
        如果是数组实现，你可能会用 二分查找 (Binary Search) —— 这是一个算法。
        如果是链表实现，你只能从头一个个找 —— 这也是一个算法（虽然比较笨）。
